```

```

→ 글로벌 분산투자 자동화에 유리. 단일 API로 6개국 자동매매 가능.

## 3. 거시경제/대체 데이터 소스 (RESEARCH.md에 누락)

워렌 버핏 스타일 가치투자에는 **거시경제 데이터**가 필수.

| 소스 | 무료 | Python | 데이터 |
|------|:---:|--------|--------|
| **FRED (미국 연준)** | O | `fredapi` | 금리, CPI, 실업률, GDP 등 87,000+ 시계열 |
| **한국은행 경제통계시스템 (ECOS)** | O | REST API | 기준금리, 소비자물가, M2 등 한국 거시 |
| **World Bank Open Data** | O | `wbgapi` | 글로벌 경제 지표 |
| **OECD Data** | O | REST | OECD 국가 비교 경제 지표 |
| **FinanceDataReader** | O | 내장 | FRED+OECD+KRX 통합 (이미 RESEARCH.md에 포함) |

### ECOS API 상세 (한국은행)
- **엔드포인트**: `https://ecos.bok.or.kr/api/`
- **인증**: 무료 API 키 발급
- **주요 통계**: 기준금리, 환율, CPI, PPI, 수출입, 가계부채
- **호출 제한**: 일 10,000회
- **버핏 활용**: 금리 사이클 → 주식/채권 배분 자동 조절

## 4. 백테스팅 프레임워크 (RESEARCH.md에 누락)

자동투자 시스템에 **백테스팅**은 필수. 전략 검증 없이 실전 투자는 위험.

| 프레임워크 | 설치 | 한국 | 특징 |
|-----------|------|:---:|------|
| **Backtrader** | `pip install backtrader` | 커스텀 | 가장 성숙. 다중 전략, 브로커 연동 |
| **Zipline-Reloaded** | `pip install zipline-reloaded` | X | Quantopian 오리진. 데일리 기반 |
| **bt** | `pip install bt` | O | 간결한 API. 포트폴리오 리밸런싱 특화 |
| **VectorBT** | `pip install vectorbt` | O | **속도 최고** (벡터화). 시각화 내장 |
| **QuantStats** | `pip install quantstats` | O | 성과 분석/리포팅. 샤프, 소르티노, 드로다운 |

### 추천 조합
```
전략 개발    → VectorBT (빠른 프로토타이핑)
정밀 백테스트 → Backtrader (이벤트 기반, 브로커 시뮬레이션)
성과 리포트   → QuantStats (Tearsheet 자동 생성)
```

## 5. 워렌 버핏 가치투자 지표 — API 매핑

오마하부자의 핵심 컨셉인 "버핏식 가치투자"에 필요한 지표와 데이터 소스 매핑:

### 5-1. 기본적 분석 지표

| 지표 | 의미 | 데이터 소스 | 버핏 기준 |
|------|------|-----------|----------|
| **PER** | 주가수익비율 | KIS API `per` / yfinance | < 15 (저평가) |
| **PBR** | 주가순자산비율 | KIS API `pbr` / yfinance | < 1.5 |
| **ROE** | 자기자본이익률 | DART 재무제표 / yfinance | > 15% (3년 연속) |
| **부채비율** | 재무 안정성 | DART / SEC EDGAR | < 50% |
| **영업이익률** | 수익성 | DART / SEC | > 15% (해자 지표) |
| **FCF** | 잉여현금흐름 | DART / SEC | 양(+), 증가 추세 |
| **EPS 성장률** | 이익 성장 | KIS / yfinance | > 10% (5년 CAGR) |
| **배당성향** | 주주환원 | KIS / yfinance | 30~50% (적정) |

### 5-2. 버핏 "경제적 해자" 스크리닝 로직

```python
# 오마하부자 1차 스크리닝 (버핏 기준)
BUFFETT_SCREEN = {
    "per": {"max": 15},           # 저평가
    "pbr": {"max": 1.5},          # 자산 대비 저평가
    "roe": {"min": 15, "years": 3}, # 3년 연속 고ROE
    "debt_ratio": {"max": 50},     # 낮은 부채
    "op_margin": {"min": 15},      # 높은 영업이익률 (해자)
    "fcf": {"positive": True},     # 양의 잉여현금
    "eps_cagr_5y": {"min": 10},    # 이익 성장
}
```

### 5-3. KIS API 현재가 응답에서 바로 사용 가능한 버핏 지표

RESEARCH.md에서 KIS API 응답 필드를 언급했는데, 버핏 분석에 직접 매핑:

```
stck_prpr  (현재가)     → 매수 가격 결정
per        (PER)        → ★ 버핏 핵심 지표
pbr        (PBR)        → ★ 버핏 핵심 지표
eps        (EPS)        → ★ 이익 기반 밸류에이션
bps        (BPS)        → 순자산가치
w52_hgpr   (52주 최고)  → 고점 대비 할인율
w52_lwpr   (52주 최저)  → 바닥 근접 여부
hts_frgn_ehrt (외국인보유율) → 외국인 수급
stck_avls  (시가총액)   → 대형주/중형주 분류
```

→ **KIS API 단일 호출로 버핏식 1차 스크리닝 가능**. 추가 재무제표는 DART에서 보충.

## 8. 리스크 & 법적 고려사항

### 8-1. 자동매매 법적 이슈

- **금융투자업법**: 개인 자동매매는 합법. 단, 타인 자금 운용 시 자산운용사 등록 필요
- **세금**: 해외주식 양도소득세 250만원 초과분 22%, 국내주식 대주주(10억) 양도세
- **KIS API 이용약관**: 상업적 재배포 금지, API 키 노출 금지

### 8-2. 시스템 리스크 관리

```
필수 안전장치:
├── 일일 최대 손실 한도 (예: -3% → 전체 매매 중지)
├── 단일 종목 최대 비중 (예: 포트폴리오의 10%)
├── 주문 실행 전 Telegram 확인 요청 (선택적)
├── 장 시작/종료 30분 매매 제한 (변동성 회피)
└── 긴급 정지 버튼 (Telegram 또는 대시보드)
```
